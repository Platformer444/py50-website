Making A New Command


To make a new command, you will have to follow the following steps:

Step 1: Remix the Py50 Code Running Test Project

First you will have to go to the Py50 Code Running Test Project. Its an experimental playground for Py50. It contains only the compiler for the Py50 Code. After you go to the project, Remix it.

Step 2: Get familiar with the playground

The playground is just Py50 but without a code editor. To add code, you have to use the "Write Code (Code)" custom block under the "Write Code" message in the Stage.

Step 3: Defining your command

Now before you code up the command, you have to define it. This is done so that the compiler can differentiate between variable configurations and commands. To define your command, you need to go to the Stage and under the the "When Green Flag Clicked" hat block, before the brodcasts, use the "Add New Command: (Command)" custom block to define your command.

Step 4: Code!

After defining your command, its time to code it. Go to the run sprite and find the "Run Code" defination. In the else of the "if <(Commands) contains (Content)?>", put an if-else and in the condition put "<(Content) = (command_name)>". Now make a new block and name it according to the command_name. Now put the call for the custom block in the if and in the else you can keep adding commands.

Step 5: Share!

Once the command is ready, you can share the project. I will check and see if it is really worth adding.

Step 6: Docs

Once your command is accepted, you will be commented to prepare the docs of your command. Fork the py50-website repository on github(You require a GitHub account for this). Once you have forked the repository, you can start creating the Docs. Once the docs is created, you can open a pull request and I will review it and merge.
